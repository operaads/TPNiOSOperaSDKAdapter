# TopOn Demo with OpAdxAdapterTopon

This demo project demonstrates the integration of OpAdxSDK with TopOn/AnyThink mediation using local Pod dependencies.

## Project Structure

```
ToponDemo_oc_6.5.34/
├── iOSDemo.xcworkspace          # Open THIS file in Xcode
├── iOSDemo.xcodeproj             # Demo app project
├── Podfile                       # Pod dependencies configuration
└── Pods/                         # Generated by CocoaPods
    ├── OpAdxSdk/                 # Local SDK from ../../../OpAdxSdk
    └── OpAdxAdapterTopon/        # Local adapter from ../../../Adapter/OpAdxAdapterTopon/OpAdxAdapterTopon
```

## Local Dependencies

This project uses local Pod paths to reference:

1. **OpAdxSdk**: `../../../OpAdxSdk`
2. **OpAdxAdapterTopon**: `../../../Adapter/OpAdxAdapterTopon/OpAdxAdapterTopon`

This allows you to:
- Modify SDK code and see changes immediately
- Modify adapter code and test changes in the demo
- Build both SDK and adapter as part of the demo workspace

## How to Build

### Prerequisites
- Xcode 13.0 or later
- CocoaPods installed
- iOS 13.0+ deployment target

### Build Steps

1. **Install Dependencies** (required after any Podfile changes):
   ```bash
   cd /path/to/Demo/ToponAdapter/ToponDemo_oc_6.5.34
   pod install
   ```

2. **Open Workspace** (NOT the .xcodeproj):
   ```bash
   open iOSDemo.xcworkspace
   ```

3. **Select Scheme**:
   - Choose "iOSDemo" scheme
   - Select your target device or simulator

4. **Build and Run**:
   - Press Cmd+R or click the Run button

## Making Changes

### To OpAdxSDK
1. Edit files in `../../../OpAdxSdk/OpAdxSdk/`
2. Changes are immediately available - just rebuild

### To OpAdxAdapterTopon
1. Edit files in `../../../Adapter/OpAdxAdapterTopon/OpAdxAdapterTopon/OpAdxAdapterTopon/CustomAdapter_6.5.xx/`
2. Changes are immediately available - just rebuild

### To Demo App
1. Edit files in `iOSDemo/`
2. Rebuild as normal

## Troubleshooting

### "OpAdxSdk module not found"
- Run `pod install` again
- Clean build folder (Cmd+Shift+K)
- Restart Xcode

### "AdapterVersion.h not found"
- Make sure you're importing via angle brackets: `#import <OpAdxAdapterTopon/AdapterVersion.h>`
- Not via relative path: `#import "../../AdDemo/CustomAdapter_6.5.xx/AdapterVersion.h"`

### Duplicate UUID warnings
- These are harmless warnings from CocoaPods and can be ignored

## Configuration Details


- OpAdxSdk: 2.8.1
- OpAdxAdapterTopon: 2.8.1.0
- TopOn SDK: 6.5.34

## Demo Features

This demo showcases all supported ad formats:
- Banner Ads
- Interstitial Ads
- Rewarded Video Ads
- Native Ads
- Splash/App Open Ads

Each ad format has a dedicated view controller demonstrating proper integration.

## Notes

- The adapter code in Demo/iOSDemo/AdDemo/CustomAdapter_6.5.xx is NOT used when OpAdxAdapterTopon pod is installed
- All adapter code comes from the Pod (../../../Adapter/OpAdxAdapterTopon/OpAdxAdapterTopon)
- Adapter files include ATSafeThreadArray_OP for thread-safe initialization callback handling
